# Learning Assignments

<!-- Content will be enriched by AI parsing scripts -->

## Overview

The Learning Assignments module manages the distribution and tracking of educational content to trainees and faculty. Programs assign learning materials -- such as articles, videos, policy documents, online modules, and quizzes -- to users with due dates and completion tracking. This module supports mandatory compliance training (e.g., HIPAA, safety), program-specific curriculum delivery, and self-directed learning.

### Key Responsibilities

- **Assignment Creation**: Create and distribute learning materials with configurable due dates, target audiences, and completion criteria
- **Completion Tracking**: Monitor assignment completion status with timestamps, attempts, and scores
- **Due Date Management**: Configurable due dates with automated reminders at multiple intervals
- **Curriculum Management**: Group assignments into curricula or learning paths for structured training
- **Quiz Integration**: Link assignments to quiz assessments for knowledge verification
- **Compliance Reporting**: Track mandatory training completion rates for compliance audits

## Business Workflows

### Assignment Distribution

```mermaid
sequenceDiagram
    participant Admin as Program Admin
    participant LAM as LearningAssignmentManager
    participant LRM as LearningReminderManager
    participant MAIL as MailManager
    participant DB as SQL Server

    Admin->>LAM: CreateAssignment(contentId, targetUsers[], dueDate)
    LAM->>DB: sp_Learning_InsertAssignment
    LAM->>DB: sp_Learning_BulkInsertUserAssignments

    LAM->>MAIL: SendAssignmentNotification(users[], contentTitle, dueDate)
    LAM->>LRM: ScheduleReminders(assignmentId, reminderIntervals[])
    LRM->>DB: sp_Learning_InsertReminders

    Note over LRM,DB: Reminder intervals: 7 days before,<br/>3 days before, 1 day before, overdue
```

### Assignment Completion Flow

```mermaid
sequenceDiagram
    participant User as Trainee
    participant LAM as LearningAssignmentManager
    participant QM as QuizManager
    participant MAIL as MailManager
    participant DB as SQL Server

    User->>LAM: StartAssignment(assignmentId)
    LAM->>DB: sp_Learning_UpdateStatus("In Progress")
    LAM->>DB: sp_Learning_GetContent(contentId)
    DB-->>LAM: Content details + URL
    LAM-->>User: Learning content

    alt Has Quiz Component
        User->>QM: TakeQuiz(quizId)
        QM-->>User: Quiz questions
        User->>QM: SubmitQuiz(answers[])
        QM->>QM: GradeQuiz(answers)
        QM-->>LAM: QuizResult(score, passed)

        alt Passed
            LAM->>DB: sp_Learning_MarkComplete(assignmentId, score)
        else Failed
            LAM->>DB: sp_Learning_UpdateStatus("Retry Required")
            LAM-->>User: Retry instructions
        end
    else No Quiz (completion on view)
        User->>LAM: MarkComplete(assignmentId)
        LAM->>DB: sp_Learning_MarkComplete(assignmentId)
    end

    LAM->>MAIL: SendCompletionConfirmation(userId, contentTitle)
```

### Compliance Monitoring

```mermaid
sequenceDiagram
    participant Scheduler as Windows Service
    participant LCM as LearningComplianceManager
    participant MAIL as MailManager
    participant DB as SQL Server

    Scheduler->>LCM: RunComplianceCheck()
    LCM->>DB: sp_Learning_GetMandatoryRequirements
    DB-->>LCM: Mandatory training requirements

    loop For each requirement
        LCM->>DB: sp_Learning_GetCompletionStatus
        DB-->>LCM: Completion status per user

        loop For each overdue user
            LCM->>MAIL: SendOverdueNotice(user, requirement)
        end

        alt Escalation threshold reached
            LCM->>MAIL: SendEscalation(programDirector, overdueUsers[])
        end
    end

    LCM->>DB: sp_Learning_UpdateComplianceSnapshot
```
